    • Project Purpose Goal

    This project aims to build an official website for Alberta Bike Swap to display general event information and act as a bike inventory display portal for buyers to speed up the purchasing process.

    Web Stack and Explanation
    • Built the project using JSP + Java Servlet because it is easy to maintain, can handle dynamic data and has better security.
    • A backend server is required, and Tomcat is the application server in this project.
    • Bike inventory data is handled through Json and a JSON API, Gson; then, data will be processed through the servlet to generate data back to the web application.

    Problem and Thought Process

    This project has undergone different development stages to meet the client's requirements. The original solution of this project is to build a management system using the Oracle database to support data processing and purchasing processes. But the client had a plan to develop a supporting system in PHP and wanted the system to be separate from the website. The major problem became how to handle data communication between the two systems.

    To reduce the communication complexity and improve data querying, using JSON is the best solution because both systems don't need to communicate with the other directly. Low coupling between both systems reduces errors and the complexity of future maintenance, especially if they are built with different languages.

    Download a PDF format poster which associate with the event's information page is implemented with javascript and jsPDF, in order to provide more convenience way for participants to share information across social media. Difficulties during building this function:

    • How the webpage has build-up doesn't fit with the common method, simply using HTML tags to get the webpage content and then converting them into PDF.
    • Different location pictures' path also need to be obtained separately due to the webpage build.
    • During the process of loading pictures, may get errors like base64 error and cross-site blocking.

    Using the onload method (img.onload = function(){}) with crossorigin Attribute crossOrigin (img.crossOrigin) to get the jfPDF obtains the desired picture to reduce extra picture conversion and cross-site blocking.

    Lessons Learned

    The PHP supporting system is unavailable during the webpage development, so we can only have a simulation for the final deliverable. However, we can still take advantage of picking some common language independence stacks for data exchange because this helps reduce development's complexity and provide better integration. This opportunity allowed me to learn more about better system design. Carefully picking an appropriate stack is critical because it may lead to total re-coding during the development cycle.
